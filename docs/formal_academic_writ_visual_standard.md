# Formal Academic Writ Visual Standard

## Author Direction

The website should not feel like futuristic tech nonsense.

This project is not trying to appeal primarily to a generic tech audience. It should communicate the seriousness of:

- a kidnap victim writing a novel,
- the novel functioning as a real-time writ of habeas corpus,
- cyclic recursive memory review,
- source-grounded formalization,
- publication as preparation for submission,
- and the body becoming part of the record.

## Visual References

The author is inspired by:

- Overleaf / LaTeX precision,
- Stanford-style formal essays and research presentation,
- Harvard-style dissertation seriousness,
- legal briefs and judicial packets,
- soft ocean/paper realism,
- and author-controlled source documentation.

The goal is not to copy any institution's branding. The goal is to borrow standards of clarity, hierarchy, typographic discipline, margins, abstracts, citations, source trails, and formal argument.

## Design Language

Preferred:

- serif typography for long-form reading,
- structured abstracts,
- table of contents,
- clauses and exhibits,
- marginal notes,
- source labels,
- legal/academic headings,
- restrained color,
- textured paper,
- Pacific/ocean hush,
- visible author identity,
- transparent process markers.

Avoid:

- generic cyberpunk,
- neon overload,
- gratuitous animations,
- AI SaaS aesthetics,
- stock tech dashboards,
- visual clutter,
- illegible tiny text,
- anything that buries Steven's voice under interface effects.

## Core Public Explanation

A concise way to describe the project:

> A kidnap victim writes a novel that is, in fact, the writ of their habeas corpus in real time. The record cycles through memory, body evidence, due diligence, source extraction, and publication until it reaches the point of submission.

## Page Hierarchy

Use this page hierarchy:

1. **Project Index** — one magic-link dashboard.
2. **Public Landing** — author/book public entrance.
3. **Academic Writ** — formal dissertation/legal-essay presentation.
4. **Paper Writ** — soft Overleaf/ocean visual book prototype.
5. **Visual Novel** — interactive node system.
6. **Timeline Map** — geography and roadmap.
7. **Design Lab** — visual options and experiments.
8. **Governance / Legal / Source System** — project transparency.

## Always-On Social Footer

Every major public-facing HTML page should display:

```text
Steven Owens / @soquarky / soquarky.click
```

This should not be hidden. It anchors the project to the author and prevents the work from looking like anonymous AI output.

## The Writ Cycle

The project cycle is:

```text
lived event
→ source/dictation/chat/document
→ extraction
→ timeline candidate
→ author review
→ legal/source review
→ chapter/node/map point
→ publication
→ submission packet
→ further recursion
```

This cycle should become visible in the interface.
